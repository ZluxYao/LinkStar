# Home 模块需求

主页（LinkStar 首页）的配置 / 数据 / API 备忘。

## 1. 模块定位

Home 只做**展示层**：

- 把 stun 内网穿透服务展示成可点击的卡片（带在线状态、三栏地址）
- 用户自定义的「常用网站」也以卡片形式展示
- 提供搜索框、壁纸、分类、布局等主页装饰能力

Home 模块**不**直接创建/管理 stun 服务。

## 2. App 的两种类型

| 类型 | 来源 | 在线状态 | 地址 |
|---|---|---|---|
| `stun` | stun 模块「也在 home 显示」开关创建 | 来自 stun 探活 | 来自 stun（WANv4 / WANv6 / LAN） |
| `static` | home 模块「添加常用网站」按钮 | 不显示绿点 | 用户填写（常用网站一般只填 WANv4） |

### 创建/删除入口

- **stun 类型**：
  - 创建：仅在 stun 模块新建 / 编辑 service 时打开「也在 home 显示」开关时联动创建
  - 删除：
    - stun 删除 service → 自动级联删除对应 home app（单向 hook）
    - stun 关闭「显示在 home」开关 → 等价于删除对应 home app
    - 在 home 删除该卡片 → 仅删卡片，stun 不受影响
  - 开关状态显示：实时查询 home 里是否存在对应 ServiceID 的 app，避免开关状态和实际 home 不一致

- **static 类型**：
  - 创建/删除/编辑 全部在 home 模块完成

### 一致性约束

- 一个 stun service 至多被一个 home app 引用（一对一）
- 删除 home stun 类型卡片不会让 stun 那边的开关回到 off 状态以外，无其他副作用

## 3. 级联（仅 stun → home 单向）

```
stun.DeleteService(deviceID, serviceID)
    └─> emitServiceDeleted(deviceID, serviceID)
            └─> home 注册的回调
                    └─> 移除所有 app.Type=="stun" && app.StunServiceID==serviceID

stun.DeleteDevice(deviceID)
    └─> 遍历 device.Services 逐个 emitServiceDeleted

home.DeleteApp(...)  → 不影响 stun
```

实现：在 `modules/stun/hooks.go` 加 `RegisterOnServiceDeleted`，home 模块 init 注册。stun 不感知 home 的存在，避免循环 import。

## 4. 数据模型（modules/home/model.go）

```go
const SearchHistoryMax = 24

type Config struct {
    Title    string `json:"title"`    // 主页标题
    ShowTime bool   `json:"showTime"` // 是否显示时间

    Wallpaper Wallpaper `json:"wallpaper"`

    LayoutMode    string `json:"layoutMode"`    // paged-horizontal / paged-vertical / paged-free / scroll
    NetworkPrefer string `json:"networkPrefer"` // wanV4 / wanV6 / lan

    DefaultSearchEngineID string         `json:"defaultSearchEngineId"`
    SearchEngines         []SearchEngine `json:"searchEngines"`
    SearchHistory         []string       `json:"searchHistory"` // 最多 24 条,服务端裁剪

    Categories []Category `json:"categories"`
    Apps       []App      `json:"apps"`

    CreatedAt time.Time `json:"createdAt"`
    UpdatedAt time.Time `json:"updatedAt"`
}

type Wallpaper struct {
    Mode       string `json:"mode"`       // default / bing
    Resolution string `json:"resolution"` // 1080 / uhd
    Blur       int    `json:"blur"`       // 0-12
}

type SearchEngine struct {
    ID        string `json:"id"`
    Name      string `json:"name"`
    ShortName string `json:"shortName"`
    URL       string `json:"url"`   // 含 ?q= 等查询前缀
    Color     string `json:"color"` // tailwind gradient,无图标时背景
    Icon      string `json:"icon"`  // /icons/xxx 或 data/icon/xxx
    Order     int    `json:"order"`
}

type Category struct {
    ID    string `json:"id"`
    Name  string `json:"name"`
    Order int    `json:"order"`

    CreatedAt time.Time `json:"createdAt"`
    UpdatedAt time.Time `json:"updatedAt"`
}

type App struct {
    ID         string `json:"id"`
    Name       string `json:"name"`
    Icon       string `json:"icon"`       // data/icon/xxx 文件路径
    Color      string `json:"color"`      // tailwind gradient,无图标时背景
    CategoryID string `json:"categoryId"` // 空 = 未分类

    PagedOrder  int `json:"pagedOrder"`  // 翻页模式全局序
    ScrollOrder int `json:"scrollOrder"` // 整页模式分类内序

    Type string `json:"type"` // stun / static

    // type=stun
    StunDeviceID  uint `json:"stunDeviceId,omitempty"`
    StunServiceID uint `json:"stunServiceId,omitempty"`

    // type=static
    Addresses *AppAddresses `json:"addresses,omitempty"`

    CreatedAt time.Time `json:"createdAt"`
    UpdatedAt time.Time `json:"updatedAt"`
}

// 常用网站只填 WANv4 一栏即可,本地服务/stun 三栏都有值
type AppAddresses struct {
    WANv4 string `json:"wanV4"` // 公网 IPv4 入口 / URL,如 https://github.com 或 http://1.2.3.4:8080
    WANv6 string `json:"wanV6"` // 公网 IPv6 全球地址(可选,无需打洞)
    LAN   string `json:"lan"`   // 内网 IPv4 入口(可选)
}
```

### 返回前端的视图（运行时合成）

`GET /home/config` 返回时，stun 类型的 app 由后端 hydrate 成统一格式，前端不感知来源差异：

```go
type AppView struct {
    ID         string       `json:"id"`
    Name       string       `json:"name"`
    Icon       string       `json:"icon"`
    Color      string       `json:"color"`
    CategoryID string       `json:"categoryId"`
    PagedOrder  int         `json:"pagedOrder"`
    ScrollOrder int         `json:"scrollOrder"`
    Addresses  AppAddresses `json:"addresses"` // 不论来源,统一三栏
    Online     *bool        `json:"online,omitempty"` // nil=不展示绿点; true/false=stun 类型才有
}
```

## 5. 配置文件

- 路径：`config/homeConfig.json`
- 风格仿 stun：`ReadConfig` / `createConfig` / `UpdateConfig` 三件套
- 首次启动 `createConfig` 写入默认值（见第 6 节）

## 6. 默认值（首次创建配置时）

| 字段 | 默认 |
|---|---|
| Title | LinkStar |
| ShowTime | true |
| Wallpaper.Mode | bing |
| Wallpaper.Resolution | 1080 |
| Wallpaper.Blur | 0 |
| LayoutMode | paged-free |
| NetworkPrefer | wanV4 |
| DefaultSearchEngineID | bing |
| SearchEngines | 必应 / 谷歌 / DDG / Brave / GitHub / 知乎 / B站（参考前端硬编码） |
| Categories | 生产力 / 工具 / 娱乐 |
| Apps | 空（用户自己加） |

## 7. 图标存储

- 目录：`data/icon/`
- 上传：`POST /home/icon/upload`（multipart），后端落盘并返回相对路径如 `data/icon/xxx.svg`
- 静态服务：gin 注册 `data/icon/` 为 static dir，前端通过相对 URL 直接拿
- App.Icon 字段存路径字符串，前端 `<img src={app.icon}>` 即可

## 8. 网络模式

- 全局偏好 `NetworkPrefer`：`wanV4`（公网 IPv4） / `wanV6`（公网 IPv6） / `lan`（内网）
- 前端点击 app 卡片时，按当前偏好从 `Addresses` 三栏里挑一个打开；当前偏好为空时回退到下一栏（如 wanV4 → wanV6 → lan）
- 主页右下角 Home 图标按钮在三种偏好间切换，调 `PUT /home/network`

## 9. 布局模式

四种之一：

| 值 | 行为 |
|---|---|
| `paged-horizontal` | 仅左右翻页 |
| `paged-vertical` | 仅上下翻页 |
| `paged-free` | 上下/左右都可（默认） |
| `scroll` | 整页纵向滚动,按分类分组,标题 + grid |

排序字段：

- 翻页模式按 `App.PagedOrder` 升序（跨分类）
- 整页模式按 `App.CategoryID` 分组后内部按 `App.ScrollOrder` 升序

## 10. API 路由（按动作拆，仿 stun 风格）

```
GET    /home/config                    # 整体读(含 hydrated apps)
GET    /home/bing-wallpaper            # 现有,不改

PUT    /home/wallpaper                 # 改壁纸
PUT    /home/layout                    # 改布局模式
PUT    /home/network                   # 改 NetworkPrefer (wanV4/wanV6/lan)

POST   /home/search-engine/add
PUT    /home/search-engine/update
DELETE /home/search-engine/delete
PUT    /home/search-engine/reorder
PUT    /home/search-engine/default     # 设默认引擎

GET    /home/search-history
POST   /home/search-history/add        # 服务端裁剪到 24
DELETE /home/search-history/clear

POST   /home/category/add
PUT    /home/category/update
DELETE /home/category/delete
PUT    /home/category/reorder

POST   /home/app/add                   # 仅 type=static
PUT    /home/app/update                # 通用,但 stun 类型只允许改 Name/Icon/Color/Category/Order
DELETE /home/app/delete                # stun/static 都允许,不影响 stun
PUT    /home/app/reorder               # body: { mode: "paged"|"scroll", ids: [...] }
PUT    /home/app/category              # 拖拽分配分类

POST   /home/icon/upload               # multipart,返回 data/icon/xxx
```

stun 模块新增（用于联动）：

```
PUT    /stun/service/show-on-home      # 切换显示开关,内部调 home.AddApp / DeleteApp
```

## 11. 前后端职责切分

| 项 | 前端 | 后端 |
|---|---|---|
| 首屏数据 | GET /home/config 拉取 | hydrate stun 类型 app |
| 添加常用网站 | 弹窗收集 → POST /home/app/add | 校验 / 落库 |
| 添加 stun 卡片 | 不在 home 操作 | 由 stun 联动 |
| 拖拽排序 | 拖完后整段提交 ids | 重写顺序字段 |
| 分类增删 | UI + API 调用 | 落库 |
| 网络模式切换 | 切换按钮 → PUT /home/network | 写 Config |
| 搜索历史 | 提交一次 → POST add | 裁剪到 24 |
| 在线状态 | 读 AppView.Online | hydrate 时取 stun 探活结果 |
